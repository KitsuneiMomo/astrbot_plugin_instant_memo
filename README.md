一款为 AstrBot 设计的 **AI 自治备忘录**插件。与传统的"用户手动设闹钟"不同，本插件让 AI 自身拥有"记事本"和"日程表"——它可以自主记录状态备忘、创建定时/周期任务、注册关键词搭话监听，且一切操作均在对话中自然完成，无需用户学习额外指令。

---

## ![diff](https://img.icons8.com/material-outlined/24/idea.png) 与传统备忘录/定时任务插件有何不同？

| 对比维度 | 传统备忘录/定时插件 | 速记备忘录 (Instant Memo) |
| :--- | :--- | :--- |
| **触发方式** | 用户手动输入指令设置 | AI 在对话中自主判断并创建 |
| **API 额度消耗** | 每次操作触发 Tool Call 多轮往返 | XML 标签模式单次请求即完成，零额外开销 |
| **定时任务内容** | 固定文本或简单模板 | 触发前动态读取上下文，由 LLM 实时渲染生成 |
| **周期任务支持** | 通常仅支持单次提醒 | 支持单次 / 每日 / 间隔循环三种模式 |
| **关键词搭话** | 不支持 | AI 可注册关键词监听，命中时结合上下文主动搭话 |
| **状态备忘录** | 不支持 | AI 可为自己设置临时人设/状态，到期自动过期 |
| **小模型兼容性** | 依赖稳定的 Function Calling | XML 模式对 7B/8B 小模型同样友好 |

## ![diff](https://img.icons8.com/material-outlined/24/idea.png) 设计考量：为何提供 XML 标签模式？

本插件同时支持 **Tool Call（工具调用）** 和 **XML Tag（标签解析）** 两种触发模式，可在配置面板中自由切换。提供 XML 标签模式基于以下考量：

1. **节约 API 额度与请求频次**
   Tool Calling 的典型流程为：*LLM 决定调用 → 框架执行 → 返回结果 → LLM 继续生成*，一次操作至少产生两轮 API 请求。XML 标签模式下，AI 仅需在常规回复文本中附带一个 `<ai_memo ... />` 标签，插件在后台正则提取并执行，全程只消耗一次 LLM 请求。

2. **极低的 Token 开销**
   Tool Calling 需要在 System Prompt 中注入庞大的 JSON Schema 工具定义（每个工具通常数百 Token）。XML 模式仅需注入一小段使用说明，Prompt Token 消耗显著降低。

3. **更广泛的轻量级模型兼容性**
   许多 7B/8B 规格的小模型 Function Calling 能力不稳定，容易出现参数解析错误或忽略调用指令。输出简单的 XML 标签对模型的指令遵循能力要求相对较低，各种模型均能保持极高的成功率。

4. **即时执行与无感体验**
   插件在 LLM 输出后立即解析 XML 标签并执行操作，同时从最终发送给用户的文本中清除标签本身，整个过程对用户完全透明。

> **提示**：如果你的模型 Function Calling 能力稳定，也可以在配置中选择 Tool Call 模式，两种模式功能完全等价。

## ![config](https://img.icons8.com/material-outlined/24/settings.png) 配置说明

插件启用后，可在 AstrBot 的 Web 控制面板中配置以下选项：

| 配置项 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `poll_interval` | `int` | `15` | 后台轮询检查定时任务的间隔（秒）。 |
| `allow_global_memo` | `bool` | `true` | 是否允许 AI 创建全局状态备忘录（关闭则仅当前会话生效）。 |
| `trigger_mode` | `string` | `tool` | 触发模式：`tool`（工具调用模式）或 `xml`（XML 标签模式）。 |

## ![usage](https://img.icons8.com/material-outlined/24/book.png) 功能详解

### 1. 状态/人设备忘录

AI 可以为自己设置临时的状态或人设备忘录，在有效期内会被注入到 System Prompt 中，过期后自动清除。

- 支持设定有效时长（分钟）
- 支持全局或当前会话作用域
- AI 可自主删除不再需要的备忘录

### 2. 定时/周期任务

AI 可以自主创建主动推送任务，在触发时刻到来前 1 分钟自动读取最新会话上下文，由 LLM 动态生成推送内容。

- **单次定时 (one_off)**：指定分钟数后触发一次
- **每日定时 (daily)**：每天固定时刻触发（HH:MM 格式）
- **间隔循环 (interval)**：每隔固定分钟数周期性触发

### 3. 关键词搭话监听

AI 可以注册关键词监听器，当群聊或私聊中出现指定关键词时，自动结合上下文生成搭话内容并主动发送。

- 支持全局或当前会话作用域
- 可配置上下文历史条数
- AI 可自主管理（创建/删除）触发器

## ![admin](https://img.icons8.com/material-outlined/24/shield.png) 用户管理指令

用户可通过以下指令手动管理所有备忘录、定时任务和关键词搭话：

| 指令 | 说明 |
| :--- | :--- |
| `/memo` 或 `/memo list` | 列出所有备忘录、定时任务与搭话监听器 |
| `/memo del <序号>` | 删除指定的条目 |
| `/memo edit <序号> <属性> <新值>` | 修改指定条目的属性 |

**可修改属性：**
- **状态备忘录**：`content`（内容）、`expire`（过期时间/分钟）、`global`（是否全局）
- **定时任务**：`desc`（描述）、`value`（触发时间）、`type`（类型: one_off/daily/interval）
- **关键词搭话**：`keyword`（触发词）、`desc`（回复设定）、`global`（是否全局）

## ![warning](https://img.icons8.com/material-outlined/24/error.png) 实现原理

1. **System Prompt 注入**：每次 LLM 请求前，插件在系统提示词末尾动态追加当前生效的备忘录状态、运行中的任务列表和已注册的关键词监听器，使 AI 始终感知自身的"备忘板"。
2. **双模式触发**：
   - **Tool Call 模式**：将备忘录操作注册为标准的 LLM 工具，由框架管理调用流程。
   - **XML Tag 模式**：AI 在回复文本中输出 `<ai_memo>` 标签，插件在后处理阶段正则提取并执行对应操作，同时从最终回复中清除标签。
3. **后台轮询引擎**：独立的 asyncio 协程以可配置的间隔轮询任务列表，在触发时刻前 1 分钟启动 LLM 动态内容渲染，到点后自动推送。失败时自带重试机制。
4. **关键词拦截**：高优先级消息过滤器实时扫描所有消息，命中关键词后异步触发搭话流程，结合会话上下文生成自然回复。
